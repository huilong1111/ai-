# AI Review

AI Review 是一个面向日常研发场景的桌面端 AI 代码审查工具。       ->下载点击

它的目标不是只生成一段“看起来像审查意见”的文字，而是尽量给出更可靠、更可执行、更方便继续交给 AI agent 处理的结构化审查结果。

## 软件定位

AI Review 适合这些场景：

- 对 GitHub 仓库或本地仓库做快速代码审查
- 在提交改动后获取结构化风险分析
- 对历史版本做比对，观察风险是变好还是变差
- 在项目持续迭代中沉淀历史记忆、热点文件和常见风险
- 把审查建议进一步整理成适合 AI coding agent 执行的提示词

它更偏向“开发工作流工具”，而不是单次对话式问答工具。

## 核心能力

### 1. 仓库审查

支持输入：

- GitHub / 远程 Git 仓库地址
- 本地 Git 仓库路径

审查时会自动完成：

- 仓库加载
- 代码 diff 解析
- 影响范围分析
- 项目画像分析
- 历史记忆检索
- 结构化审查结果生成

### 2. 结构化审查结果

审查结果不是只有一段总结，而是拆成多个清晰模块，包括：

- 总览
- 建议栏目
- 详细审查报告
- 项目画像
- 历史记忆 / 历史风险
- 审查可靠性

重点关注：

- 潜在缺陷
- 安全风险
- 技术债
- 重构建议
- 架构演进建议

### 3. 审查结果可靠性评分

AI Review 不只依赖模型“自我感觉良好”，还会做本地质量评价。

会检查：

- 是否空泛
- 是否引用文件、路径、模块或位置
- 是否有可执行建议
- 是否能定位到具体问题
- 是否覆盖关键栏目

最终会给出：

- 审查质量分
- 质量等级
- 正向信号
- 质量警告
- 改进建议

### 4. 项目房间与历史版本

每次审查都会沉淀成项目房间里的一个版本，支持：

- 查看历史版本
- 重命名版本
- 删除版本
- 删除项目
- 自动记住项目仓库地址

这样同一个项目可以持续积累审查记录，而不是每次重新开始。

### 5. 版本比对

支持：

- 自动对比“本次审查 vs 上一次审查”
- 手动选择两个历史版本进行比对

可比较的内容包括：

- 评分变化
- 风险等级变化
- 审查质量变化
- 问题数量变化
- 建议变化
- 历史风险变化

适合观察一个版本到底是在变好，还是在引入新的风险。

### 6. AI 计划建议

AI Review 可以把结构化审查结果进一步整理成“适合 AI agent 执行”的提示词。

这个模块适合继续交给：

- Codex
- ChatGPT
- 其他 AI coding agent

用来辅助后续：

- 修复问题
- 做局部重构
- 执行删减建议
- 推进架构演进

### 7. 自动审查

对远程 GitHub 项目可以开启自动审查。

开启后，软件运行期间会定期检查远端仓库最新提交；如果发现新 commit，会自动触发一次新的后台审查，并生成新的历史版本。
